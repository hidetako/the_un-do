# 坐雲堂 YouTubeライブ 配信手順書（別PCでの立ち上げ用）

> 2026-07-02 の初回配信立ち上げで実際に迷った箇所を踏まえた、つまずかないための手順。
> 所要時間：初回セットアップ 20〜30分。2回目以降は「E. 配信開始」だけ（1分）。

---

## A. 事前に手元に用意するもの
- YouTubeアカウント（チャンネル `@Issoku_The_un_do`／ID `UCqzoMItMT74uC--y67S1-AQ`）にログインできること
- **ストリームキー**：YouTube Studio → 左メニューの電波アイコン「ライブ配信」→ 画面右上「ストリームキー: Default str…」の**右にある ⧉（コピー）アイコン**でコピー
  - ⚠️ **キーは秘密**。チャットや配信画面に写さない。漏れたら同じ画面の「新しいストリームキーを作成」で再発行
  - ⚠️ その隣の **🔄 はページ更新ではなく「キーのリセット」**。押すと配信が切れる。**ページ更新は F5**
- 回線：**有線LAN推奨**（24時間配信はWi-Fi非推奨）

## B. OBSのインストールと配信先設定
1. https://obsproject.com/ja/download から OBS Studio をインストール → 起動
2. 初回の「自動構成ウィザード」が出たら：「配信のために最適化」→ サービス **YouTube** → 「ストリームキーを使用」でキーを貼る → 適用（**これで配信先設定は完了。手順3は不要**）
3. ウィザードを使わない場合：**右下「コントロール」枠の「設定」ボタン**（メニューの「ファイル→設定」でも同じ）→ 左の「**配信**」
   - サービス：「**YouTube - RTMPS**」を選ぶ（**「カスタム」やサーバーURLの手入力は不要**）
   - 「ストリームキーを使用」→ キーを貼り付け → OK
4. 同じ設定画面で画質を確認：
   - 「**出力**」：出力モード「詳細」→ レート制御 **CBR**／ビットレート **6000 Kbps**（回線が細ければ **4500**）／**キーフレーム間隔 2 s**
   - 「**映像**」：基本・出力とも **1920×1080**／**30 fps**

## C. ソース（映すもの）の設定 ← 一番つまずいた所
1. 下の「ソース」枠の **＋ → 「ブラウザ」** → 新規作成
2. プロパティで：
   - **URL**：`https://the-un-do.vercel.app/?live=rotate&bell=30`
     （⚠️ 既定のままだとOBSの案内ページが映る。**URLを必ず入れ替える**）
     - 固定座処にしたい場合は `?live=honden` / `kare` / `umi` / `tsuki` / `neko` / `cho`
   - **幅 1920／高さ 1080**
   - ✅ **「音声をOBSで制御する」にチェック** ← ⚠️ **これを忘れると無音配信になる**（ミキサーに坐雲堂の音声チャンネルが現れるのが正解）
   - OK
3. ソースを**右クリック → 「対話」** → 開いた画面で「**▶ 坐をはじめる**」をクリック → 閉じる
   （音声のロック解除。これを押すまで音は鳴らない）
4. 「音声ミキサー」で確認：
   - **ブラウザソース（https://th…）のメーターが動いている**こと
   - **「デスクトップ音声」「マイク」はミュート（赤）のままでOK**（PCの通知音などが配信に入らない）

## D. YouTube Studio側の設定（初回のみ）
1. Studio → ライブ配信ダッシュボード。**画面が狭いと編集パネルが隠れる**ので、ウィンドウ最大化＋Ctrl＋マイナスで縮小表示にする
2. 配信情報（タイトル・説明・サムネ・カテゴリ）を「編集」から設定：
   - タイトル：`【ライブ】坐雲堂 — 一緒に坐る／鐘と環境音 ｜ zazen "sit with me" live`
   - 説明：`youtube.md` のテンプレ／サムネ：`assets/youtube/thumbnail.jpg`
   - 視聴者：「**いいえ、子ども向けではありません**」
   - ⚠️ 「ライブ配信の方法」が**グレーアウトしているのは正常**（作成後は変更不可の仕様。ストリーミングソフトウェアになっていればOK）
   - ⚠️ 編集ダイアログの「公開設定：不明」も仕様。**公開範囲はダッシュボード側**（または「コンテンツ」→ライブ配信タブ）で変更する
3. 「ライブ配信の設定」欄：
   - 遅延：「**通常の遅延**」／**DVR有効=ON**
   - **自動開始=ON**（「ライブ配信を開始」ボタンを探さなくて済む・確実）
   - **自動終了=OFF**（24時間配信の生命線。ONだと回線が一瞬切れただけで配信自体が終了する）

## E. 配信開始
1. OBS右下「コントロール」の「**配信開始**」をクリック
2. OBS下部ステータスバーを確認：**緑 ■＋6000kbps前後＋ドロップ 0%＋30/30 FPS** なら正常
3. Studioに戻ると30秒〜1分で「データなし」→プレビューに坐雲堂が映る
   - ⚠️ **「ストリーミングを準備しています」のくるくるが続いても慌てない**。左上が「非常に良い」なら受信は正常。**F5でページ再読み込み**（🔄は押さない）
4. 自動開始ONなら、データが届いた時点でライブになる。確認は新しいタブで
   `https://www.youtube.com/@Issoku_The_un_do/live` を開く（映像と**音**を必ず確認）
5. 公開範囲が「限定公開」なら、確認後に「コンテンツ」→ライブ配信タブ→公開設定→「**公開**」へ

## F. 24時間運転の注意
- PCの**スリープ・休止・スクリーンセーバーをオフ**
- **Windows Updateの自動再起動**に注意（アクティブ時間の設定推奨）
- 裏で動く**クラウド同期（Dropbox等）は一時停止**推奨（上り帯域を食う）
- 「要改善（YouTubeが受信している動画が少ない）」警告は接続直後に出がち。**継続するならビットレートを4500へ**
- Googleの「通常と異なるトラフィック」ブロックが出たら：**VPN／Cloudflare WARP（1.1.1.1）をオフ**にして再読み込み（配信のRTMP送信は別経路なので無事）

## G. アプリを更新（マージ）したとき、配信に反映する
1. OBSのブラウザソースを**右クリック → 「現在のページを再読み込み」**
2. **右クリック → 「対話」 → 「▶ 坐をはじめる」をもう一度クリック**（音の再開）
- 配信は切れない。この2手順を忘れると配信画面が古いままになる

## H. つまずき早見表
| 症状 | 原因と対処 |
|---|---|
| OBSに「ファイル→設定」が見つからない | 右下「コントロール」枠に「設定」ボタンがある |
| サーバーURLをどこに入れるか分からない | 「YouTube - RTMPS」を選べば**サーバー入力は不要**。キーだけ貼る |
| プレビューがOBSの案内ページのまま | ブラウザソースの**URLが既定のまま**。プロパティでURLを入れ替える |
| 配信が無音 | ①「音声をOBSで制御する」未チェック ②「対話」で▶を押していない |
| Studioのプレビューがくるくるのまま | 「非常に良い」なら受信は正常。**F5**で再読み込み（🔄はキーのリセットなので押さない） |
| 「ライブ配信の方法」がグレーアウト | 正常（作成後は変更不可）。ストリーミングソフトウェアならOK |
| 「ライブ配信を開始」ボタンが見つからない | 設定の「**自動開始=ON**」にしてOBSを配信終了→再開すれば、ボタン不要でライブになる |
| 「要改善」警告 | 一時的なら放置で戻る。続くならビットレート4500・有線化・同期アプリ停止 |
| Googleの「通常と異なるトラフィック」 | VPN/WARP起因。オフにして再読み込み。配信自体は無事 |
