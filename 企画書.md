# オンライン座禅寺「一息（Issoku）」企画書

> 一息（Issoku）— 24時間、世界中の誰もが自由に出入りできるインターナショナル座禅寺。その中に寺「坐雲堂（The Un-do）」がある。
> 参考: オンライン自習室（study with me）スタイルの常時接続ライブ空間を、座禅（瞑想）向けに再設計する

---

## 1. コンセプト

### 一言で
**「世界のどこにいても、ログインすれば“いつもそこに座っている人がいる”オンライン座禅堂」**

自習室ライブ配信が「一人だけど、みんなと一緒に勉強している感覚」を生むのと同じ構造を、座禅に応用する。
時差を逆手に取り、地球の自転に合わせて常に世界のどこかが起きている＝**堂が一度も空（から）にならない**状態を作る。

### 3つの柱

| 柱 | 内容 |
|---|---|
| **常時開放 (Always Open)** | 24時間365日、予約不要。思い立ったらすぐ座れる。「開いている安心感」そのものが価値。 |
| **自由出入り (Come & Go)** | 入退室自由。3分でも3時間でもよい。ログイン・カメラ・発話すべて任意。沈黙が基本作法。 |
| **国境なし (Borderless)** | 宗派・国籍・宗教を問わない。多言語UIと非言語の所作（鐘・呼吸・姿勢）で言葉の壁を越える。 |

### 大切にしたい思想
- 座禅は「成果を出す場」ではなく「ただ座る（只管打坐）」場。**ゲーミフィケーション・競争・ランキングは入れない。**
- 監視ではなく**気配**。誰かがそこにいる、という静かな共在感だけを伝える。
- 「正しい座禅」を強制しない。椅子でも寝たきりでも、宗教でも単なる休息でも歓迎する。

#### 設計上の最重要論点（決定済み）：「手放す」を背骨に、庭は「作務」に

> この寺が初心者向けに**進捗・記録**を持つか、禅本来の**手放す（只管打坐）**を貫くかは、機能ではなく**思想の選択**。両者は根で矛盾する。
>
> - **方便（ほうべん）の立場**: 記録や育つ庭は初心者を入口に立たせる手段。最終的に手放される前提なら正当。
> - **手放すの立場**: 積み上げ・連続日数・スコアは、座禅が手放すべき自我（もっと／続けた／達成した）を太らせる。マインドフルネス系アプリの定番＝禅とは真逆。
>
> **【決定】**「**手放す**」を背骨にする。理由は (1) 思想的に筋が通る (2) 「何も記録しない寺」は他にない最大の差別化。
> 「荒れた庭→整う庭→枯山水」の着想は捨てず、**「記録（貯める）」ではなく「作務（さむ＝掃き清める修行）」**として活かす:
> - 庭は**生涯スコアではなく「今日の庭」**。整えても砂紋は**また掻き消される**（枯山水＝無常）。
> - **保存しない・連続日数を出さない・サボっても罰しない**（罪悪感の演出も自我を煽るので不可）。
> - 整え終えた庭が最後に**円相（まるい空）**へ溶けて消える＝「到達＝手放し」を体で示す。
> - 記録したい人向けの控えめな状態保存は**端末内のみ**（サーバーに送らない）＝「記録しない寺」を崩さない。

## 2. ターゲットユーザー

- **時差を抱える国際的な実践者** — 深夜や早朝、近所に座る場所がない人。
- **初心者** — お寺に行く敷居が高い／作法が分からない人。多言語ガイド付き。
- **経験者・僧侶** — 進行役（後述「当番」）として参加。
- **「一人だと続かない」層** — 自習室文化と同じく、ゆるい他者の存在で習慣化したい人。
- **オフィスワーカー／休憩時間の活用層** — デスクや会議室で**椅子座禅を3〜5分**。床に座れない職場でも、休憩・気分転換・集中前のリセットとして気軽に。マインドフルネス研修の受け皿にも。

## 3. コア機能（MVP）

### 3.1 常時稼働の「堂（メインルーム）」＝坐雲堂
- メインルーム「堂」の名は **坐雲堂（The Un-do）**（§10）。「一息」入れてここに入る。
- ログインすると即「堂」に入る。中央に**蝋燭/香炉の灯り**と**現在の入堂人数**。
- 各参加者は顔出し不要。**「座っている人型のアイコン」**や**「灯り（ひとつの炎＝一人）」**でだけ表現。
- 音声は基本ミュート。共有されるのは**環境音（雨・風・蝉・薪の爆ぜる音など）**のみ。

### 3.2 鐘（ベル）と進行
- 座禅の伝統に倣い、開始・終了を**鐘（鏧子/けいす）**の音で知らせる。**鐘３回＝開始 / 鐘１回＝終了**（合図ルールは §3.7）。
- 自動進行サイクル。常時稼働の堂はこのサイクルで回る。基礎は**円覚寺・居士林の坐禅会形式**に倣い「**15分坐る → 鐘 → 短い説明・休憩 → 繰り返し（×3）→ 最後に読経（任意）**」。長く坐れる人向けに「25分坐＋経行（きんひん）」サイクルも選べる。
- 「次の鐘まであと◯分」を表示。途中参加でもサイクルに乗れる。

### 3.3 個人タイマー — 線香（一炷／いっちゅう）で時を計る ★趣向

- 共同サイクルとは別に、自分用タイマー（5/15/25/45分・カスタム）。終了時に自分にだけ鐘が鳴る。
- **時間の表現を数字のカウントダウンではなく「一本の線香」にする。** 設定時間＝線香一本。
  - 時間が進むほど線香が少しずつ**短くなる**。
  - 火点（ひてん）が**ほのかに赤く灯り**、細い煙が**ゆらゆら立ちのぼる**。
  - 灰がそっと積もっていく。
- 燃え尽きて受け皿に達した瞬間＝**終了の鐘が一つ**鳴る。
- **由来**: 古来、坐禅の一回（**一炷＝いっちゅう**）は文字どおり線香一本が燃える時間で計られた。残り時間の数字より、煙と火種でゆるやかに時を感じる方が坐禅らしく、§1の「競わせない・静けさ」の理念にも合う。
- **座処と連動**: どの座処にも線香を重ねて表示でき、専用の座処「**線香一炷の前**（ほの暗い床の間に一炷）」も用意（§3.6）。「調息」の揮毫額や猫の横にも、そっと一炷を添えられる。
- **配慮**: 数字を見たい人向けに**控えめな残り時間表示**もオプションで。煙のアニメは `prefers-reduced-motion` で穏やかに／静止に。
- **香りの趣向（任意）**: 白檀・沈香など、煙の色味や立ちのぼり方の異なるプリセットを選べる（画面は視覚と名前のみ。香りは各自の実物でどうぞ）。

### 3.4 多言語ガイド（非同期）
- 「座禅のはじめ方」を多言語で用意（→ 詳細は **§5 多言語対応**）。
- 姿勢・呼吸・手の組み方（法界定印）を**イラスト/アニメ**で説明（言葉に頼らない）。
- 音声ガイド（誘導瞑想）はON/OFF自由・言語選択可。

### 3.5 静かな共在感
- 入退室は通知しない（音で乱さない）。人数の増減だけ静かに反映。
- 「今、12カ国 34人が座っています」のような**世界地図上の灯り**表示（位置はおおまかに、匿名で）。

### 3.6 座処（ざしょ / Seat）— シチュエーション選択 ★体験の核

堂はひとつの固定空間ではなく、**「どこに座るか」を一人ひとりが選ぶ**。選んだ座処が、その人の画面の**背景・環境音・光・気配**を決める。
自習室配信が背景の多彩さで没入を生むのと同じ仕掛けを、寺の情景で実現する。**絵と音で成立し言葉が要らない**ため、§5.2「翻訳しない設計」を最も体現する機能。

#### 座処カタログ（初期案）

| 分類 | 座処 | 情景・要素 |
|---|---|---|
| **堂内（聖域）** | 本堂 | 薄暗がり、蝋燭の揺らぎ、香の煙、木魚の余韻、深い静寂 |
| | 阿弥陀如来の前 | 金色の仏像を正面に、燭台の灯り、荘厳な静けさ |
| | 龍の屏風絵の前 | 墨の雲龍図、畳、張り詰めた緊張感（禅寺の方丈） |
| | **「調息」の揮毫額の前** ★原点 | 筆で揮毫した「調息（ちょうそく）」の扁額（へんがく）を正面に、簡素な床の間、墨の香り。**着想の原点＝常楽寺（上田市）本堂の「調息」扁額**（発案者が幼少から見て育った一枚／→ §下「原点とオリジナル制作」）。**調身・調息・調心の三額**にも展開可 |
| | **線香一炷（いっちゅう）の前** | ほの暗い床の間に一本の線香。火点がほのかに赤く灯り、細い煙が立ちのぼる。**線香が時間とともに短くなり、燃え尽きると終了**（タイマー連動＝§3.3） |

| **自然** | 朝の庭（四季あり） | 朝光、鳥のさえずり。**季節で桜→新緑→紅葉→雪**に変化 |
| | 夜の月 | 月明かり、虫の音、藍色の闇、静かな涼しさ |
| | 雨の日 | 軒の雨音、しっとりした空気、灰色の静けさ |
| | 山 | 稜線と霧、風の音、遠い鳥、清冽な空気 |
| | 海辺の岸壁 | 寄せる波、潮風、水平線、広がりと孤独感 |
| **ぬくもり（共にいる）** | **寝ている猫の横** | 縁側や畳で猫が丸くなって眠る。かすかな寝息・ときどきの寝返り・小さな喉鳴り（ゴロゴロ）。生きものの温もりと安心感。**猫のゆっくりした寝息が自然な「調息」のお手本**にもなる |

#### 各座処が持つレイヤー（データ設計）

座処は**拡張可能なデータ**として定義し、後から座処を追加してもコード改修が不要な構造にする。

```
Seat {
  id, name(i18nキー),
  visual:   背景イラスト / 微パララックス,
  particles: 桜・雪・雨・波しぶき 等（任意）,
  soundscape: 環境音セット（水音・雨・波・風・鳥・鐘の余韻）,
  light:    暖色の灯り / 朝光 / 月光 / 曇天,
  seasonal: true/false（四季変化の有無）,
  timeOfDay: fixed("night") | localSync（任意）
}
```

#### 動的レイヤー

- **四季** — 実日付で自動 or 手動選択。「朝の庭」が春夏秋冬で表情を変える（散る花びら・蝉・落葉・雪）。
- **時刻** — 任意でユーザーの現地時刻に同期（朝の庭は実際の朝に明るい）。「夜の月」など固定の座処はそのまま。
- **天候** — 「雨の日」は意図して選ぶ座処。フェーズ2で**実際の現地天気に連動**させる案も。

#### 共同性との両立（設計の妙）

- 座処は**個人ごとに自由**だが、鐘とタイマー（§3.2）は**全員共通**。
- つまり **「みんな同じ瞬間に座っているが、それぞれ違う情景の中にいる」**。一体感と個の静けさが両立する。
- 任意で「いま海辺の岸壁に 5人」のように座処ごとの灯りの数をそっと表示してもよい。

#### 制作・配慮

- **画風**: 和モダン、墨・生成り・灯りの橙。動きは最小（穏やかなパララックスと粒子）。
- **軽量化**: 軽い座処はSVG/CSS、描き込む座処はプリレンダ画像＋微アニメ。`prefers-reduced-motion` で静止画フォールバック。
- **宗派配慮**: 阿弥陀如来（浄土系）・雲龍図（禅宗）など宗派色のある座処が混在するのは「宗派を問わない開かれた寺」という理念の体現。描写は敬意をもって、将来は実在寺院や絵師の協力で座処を増やす。
- **「調息」扁額の原点とオリジナル制作【決定】**: この座処の着想は**常楽寺（上田市）本堂の「調息」扁額**＝本企画の精神的な原点。ただし**常楽寺への許諾交渉は行わない**（負担が大きいため）。よって **(c) 著作権に触れないオリジナルの「調息」揮毫額を自作**する。
  - **常楽寺の写真・筆跡は複製もトレースもしない** — 独自の筆（または書家への新規依頼）で「調息」を書き起こし、別個の書として制作する。
  - 原点は**私的な着想・敬意**として心に留め、対外的に「常楽寺の額」とは称さない。
  - 制作物例: `assets/chosoku-plaque.svg`（オリジナルの扁額イメージ・第一案）。

### 3.7 オリエンテーションとセッションの流れ ★初心者の入口

座る前に、流す／流さないを選べる**短い音声・字幕のオリエンテーション**で、座り方を一から導く。
初めての人でも作法に迷わず座れることを最優先にする。

> **構成の背骨は坐禅の三要「調身（ちょうしん）→ 調息（ちょうそく）→ 調心（ちょうしん）」**。からだを調え（姿勢）、息を調え（呼吸＝調息）、おのずと心が調う、という順序で台本を組む。**呼吸の案内には一貫して「調息」の語を用いる。**

#### セッションの流れ

```
① 姿勢を選ぶ                  [ 床座禅（結跏趺坐/半跏） / 椅子座禅 ]
        ↓
② オリエンテーションを流す？  [ 流す / スキップ ]
        ↓
③ 座る時間を選ぶ              [ 5 / 15 / 25 / 45分 / カスタム ]
        ↓
④ （流す場合）選んだ姿勢に応じたオリエンテーション再生 … 座り方を案内
        ↓
⑤ 鐘 ３回（チーン・チーン・チーン）＝ 座禅開始の合図
        ↓
⑥ 静かに座る（画面は座処の情景と最小限の表示のみ）
        ↓
⑦ 時間が来たら 鐘 １回 ＝ 終了の合図
        ↓
⑧ ゆっくり身体をほぐし、合掌一礼。退室は自由。
```

> **姿勢は2種類から選べる**。床に座る伝統的な座禅（結跏趺坐／半跏趺坐）と、膝や脚が痛い人・高齢の人・床に座れない環境でもできる**椅子座禅（いす坐禅）**。台本も姿勢ごとに用意する。

#### オリエンテーション台本 A：床座禅（日本語・正典）

> ※ 多言語化は §5.4 に従い、各言語の音声＋字幕を用意。固有語（坐布・結跏趺坐・丹田）は原語＋短い説明を添える。
> ※ 落ち着いた声でゆっくりと。各動作の間に「間（ま）」を取る。

```
これから座禅を始めます。お好きな座処で、楽な姿勢を取ってください。

まず、坐布（ざふ）の形を整えます。
座る方向に向かって、合掌し、ひとつ礼をします。
次に、反対を向いて、もう一度合掌し、礼をします。

坐布に腰を下ろし、両手を膝の上に置いて、下半身の力を抜いていきます。
右の腿（もも）の上に、左の腿をのせ、結跏趺坐（けっかふざ）を組みます。
※ 難しければ、左右どちらかをのせる半跏（はんか）でも、椅子に座る形でも構いません。

両肩をすっと持ち上げ、息を吐きながら、すとんと落とします。
からだを左右に、何度かゆっくり傾けて、ちょうど真ん中を探します。

星空を見上げるように、頭を天井へ向け、
首から腰にかけての力を抜きながら、ゆっくりと正面へ戻します。

目は、閉じません。けれど、見開きもしません。半分だけ、そっと開けておきます。

——ここから、息を調えます。これを「調息（ちょうそく）」といいます。
息を、鼻から吸って、口から吐く。自分のペースで、数回くりかえし、息をゆっくり調えていきます。

肩と耳、鼻と丹田（たんでん／臍の下にある、気のセンター）が、まっすぐ一直線になるように。

……それでは、鐘が三つ鳴ったら、座禅の始まりです。
もう一度、鐘が鳴るまで、ただ座ってください。
```

#### オリエンテーション台本 B：椅子座禅（日本語・正典）

> 出典に基づく作法 → 曹洞宗 曹洞禅ネット「いす坐禅のきほん」ほか（§5.6 で各言語監修）。
> ポイント: 浅く腰かけ背もたれに寄りかからない／坐骨に上体をのせ骨盤を立てる／足裏は床にぴたり。

```
これから、椅子座禅（いす坐禅）を始めます。背もたれのある椅子でも構いません。

はじめに、肩を回し、首をゆっくりほぐします。

椅子には、浅く——座面の半分より手前に腰かけ、背もたれには寄りかかりません。
両足は肩幅にひらき、足の裏を床にぴたりとつけます。膝は、座面と同じか、少し低いくらいに。

お尻の下の骨——坐骨（ざこつ）——に上体をのせ、骨盤をまっすぐ立てます。
背骨を、下から一つずつ積み上げるように伸ばし、頭をその上に、そっとのせます。

手は、右の手のひらを下に、その上へ左の手を重ね、両の親指を軽く合わせて、まるい輪をつくります（法界定印）。
それを、お腹の前、脚の付け根のあたりに、置きます。
※ 楽にしたければ、両手をそのまま膝の上に置くだけでも構いません。

からだを左右に、何度かゆっくり傾けて、ちょうど真ん中を探します。

口を閉じ、舌の先を、上の歯ぐきの裏に、そっとつけます。
目は、閉じず、見開かず、半分だけ開けて、一・五メートルほど先の床に、ぼんやりと視線を落とします。

——ここから、息を調えます。これを「調息（ちょうそく）」といいます。
息は、鼻から、自然に。ただ出入りにまかせながら、すこしずつ、ゆっくりと調えていきます。

肩と耳、鼻とおへその下（丹田）が、まっすぐ一直線になるように。

……それでは、鐘が三つ鳴ったら、座禅の始まりです。
もう一度、鐘が鳴るまで、ただ座ってください。
```

> 終わりの作法（共通）: 終了の鐘が鳴ったら、からだを左右にやさしく揺らしてほぐし、ゆっくり立ち上がる／退室する。

#### 鐘の合図（共通ルール）

| 合図 | 意味 |
|---|---|
| 鐘 **３回** | 座禅の**開始** |
| 鐘 **１回** | 座禅の**終了** |

- 鐘の音は全言語共通の非言語サイン（§5.2）。文字が読めなくても始まりと終わりが分かる。
- オリエンテーションをスキップした場合は、時間選択の直後に鐘３回で開始。

#### 数息観（すそくかん）モード ★任意・初心者の呼吸の拠りどころ

「呼吸をどうすればいいか分からない」という初心者最大のつまずきへの、古典的な処方。**調息（§3.7）の延長**として任意でON。

- ひと息ごとに **一・二・三……十** と数え、十までいったら**また一に戻る**だけ。雑念で分からなくなったら、また一から。
- 表示は極小・非言語: 呼吸アニメ（拡縮する円）に**数の点が静かに灯る**程度。声の小さなカウントもON/OFF可。
- **猫の寝息・線香の煙・波**と歩調を合わせれば、自分で数えなくても自然に整う（§3.6連動）。
- あくまで**入口の方便**。慣れたら数を手放し、ただ座る（只管打坐）へ——と一言だけ添える（§1の決定と整合）。

### 3.8 YouTube禅堂への入り方・出方（IN / OUT）★わかりやすさ優先

参考動画と同じく、座処の様子を**YouTubeライブ配信**でも開く（運営の現実と役割分担は §7.4）。アプリに不慣れな人でもYouTubeなら入れる、を狙う。
誰でも迷わないよう、**図解つきの超やさしい手順**を `/guide` に用意する。

#### 入り方（IN）

```
1. 「禅堂に入る」リンク（YouTubeライブ）を開く
2. ログインもチャットも不要。そのまま見るだけでOK
3. 画面を全画面にして、明るさを少し下げる
4. 通知をオフ／サイレントにする（途中で音に乱されないため）
5. 端末を正面に置き、座処の情景を眺める
6. オリエンテーションを聞く（流す設定のとき）
7. 鐘が３回鳴ったら、一緒に座りはじめる
```

#### 出方（OUT）

```
1. 終了の鐘（１回）が鳴ったら、ゆっくり目を開ける
2. からだを左右にやさしく揺らして、足をほぐす
3. 合掌して、ひとつ礼をする
4. そっとタブ／配信を閉じる
   ※ 途中で出ても構いません。出入りはいつでも自由です。
```

- **超重要な原則を最初に明記**: 「出入り自由・予約不要・ログイン不要・途中退室OK」。敷居を徹底的に下げる。
- 文章だけでなく**3〜4枚の図（ピクトグラム）**で示し、§5の全言語で読めるようにする。
- アプリ版（§3.1の堂）とYouTube版の**両方の入口**をトップに並べ、好きな方から入れる。

### 3.9 すきま坐禅（休憩モード）★椅子座禅が活きる入口

椅子座禅は床も道具も要らないので、**オフィスの休憩・授業の合間・家事の合間**に最適。
「お寺に行く」でも「正座する」でもない、**日常のすきま3分**から入れる軽い導線を用意する。

- **ワンタップ開始** — 姿勢=椅子・時間=3分（または5分）が既定。設定なしで即スタート。
- **控えめ表示** — 仕事中でも気まずくない地味なUI（小窓・ダーク背景・通知を出さない）。イヤホン前提で鐘も小音量。
- **超短時間プリセット** — 1 / 3 / 5分。「ひと呼吸」だけの30秒モードも。
- **再開のしやすさ** — 直前の座処・姿勢・時間を記憶し、次回はワンタップで同じ環境に戻れる。
- **休憩坐処** — §3.6の座処のうち、机に置いても馴染む静かな情景（雨の日・夜の月・枯山水など）を「休憩向け」として提案。
- **習慣化（非競争・手放す）** — 続けた手応えは「今日の庭」(§4) の作務でそっと感じる程度に留める。**回数・連続日数は出さない／保存は端末内のみ**（§1の決定に従う）。

> ねらい: 「24時間いつでも」の価値は、深夜の実践者だけでなく**昼間の数分**にも効く。椅子座禅×すきま時間で、間口を最大化する。

### 3.10 参考にする実在モデル：鎌倉・円覚寺（臨済宗大本山）

円覚寺は一般の人やオンラインへの坐禅の開き方で先を行っており、本企画の良い手本になる。**学ぶ点**と、自習室文化に合わせて**あえて変える点**を分けて整理する。

#### 学ぶ点（取り入れる）

| 円覚寺の実践 | 本企画への反映 |
|---|---|
| 初心者歓迎を徹底し、YouTube指導動画「**はじめての坐禅**」を公開 | §3.7 のオリエンテーション動画方針はこれに倣う |
| **居士林（こじりん）**＝在家（一般の人）のための坐禅道場 | 「誰でも入れる堂」の思想的モデル。名称・理念の参考に |
| 坐禅会の構成：**15分×3回・間に説明・最後に読経** | §3.2 の共同サイクルの基礎に採用（長すぎず初心者にやさしい刻み） |
| **暁天坐禅会**（毎朝6時の定時坐） | 当番制の「定時会」(§4) に活用。世界の各タイムゾーンを“暁天”が巡る運用へ |
| **志納（しのう／志納箱）**＝任意のお布施 | §8 の寄付モデルと一致（サブスクではなく喜捨） |
| 団体不可・**個人参加が原則** | 静けさと個の実践を守る運営方針の裏づけ |

#### あえて変える点（本企画の独自性）

- 円覚寺のオンライン坐禅会は **Zoomで全身を映す・定刻までに入室・途中参加不可**と規律的。
- 本企画は逆に、自習室文化に倣って **出入り自由・顔出し不要・予約不要・途中退室OK** を基本にする。
- **両立させる**：① いつでも入れる「**常時開放モード**」(§3.1) と、② 円覚寺型の規律ある「**定時の当番会**」(§4) の**両モード**を用意し、利用者が選べるようにする。初心者は気軽に常時開放へ、深めたい人は当番会へ。

## 4. 拡張機能（フェーズ2以降）

- **当番（Host）制度** — 僧侶やベテランが進行役として鐘を鳴らす定時セッション。
- **法話・提唱アーカイブ** — 短い話を多言語字幕付きで。
- **静かなチャット** — セッション後の数分だけ開く「お茶の間」。常時は開かない。
- **YouTube/配信連携** — 参考動画のように堂の様子をライブ配信（プライバシー配慮の抽象映像）。
- **「今日の庭」（作務／記録ではない）** ✅採用 — §1の決定に従い、生涯スコアではなく**その日だけの枯山水**。座る前にひと掃きして整え、終われば砂紋は**掻き消される**（無常）。保存は**端末内のみ**。連続日数・罰則なし。整え終えた庭は最後に**円相（まるい空）**へ溶ける。
- **オフライン寺との連携** — 実在の禅寺・瞑想センターと提携し、オンライン↔リアルの橋渡し。
- **音のお布施** ✅本採用（審査必須） — 利用者が自分の土地の環境音（雨・波・鳥・街の静けさ）を録って寄付し、座処の環境音にする。**お金がなくても参加できる国際的な喜捨**であり、世界と繋がる実感を生む本企画の目玉のひとつ。
  - ⚠ **何が投稿されるか分からないため、必ず審査ゲートを通す（本採用でも省略不可）**: 自動チェック（音量・長さ・無音率・既知の不適切音）＋**人による監修（管長/運営の承認）後にのみ公開**。
  - 権利処理（投稿者の同意・第三者の声や著作物が混入しないこと）、位置情報の匿名化、撤回手段を用意。
  - 段階導入: まず**運営が用意した少数の音から開始** → 審査体制が整い次第、投稿受付を開放。公開済みの音にも通報・取り下げの導線を残す。

## 5. 多言語対応（国際化 / i18n）詳細設計

「インターナショナル」を名乗る以上、ここが心臓部。方針は **「翻訳に頼りすぎず、まず言葉が要らない設計にする。その上で言葉の部分を丁寧に多言語化する」**。

### 5.1 対応言語（フェーズ別）

| Tier | 言語 | 投入フェーズ |
|---|---|---|
| **Tier 1** | 日本語 / English | フェーズ0（最初から） |
| **Tier 2** | 简体中文 / 繁體中文 / 한국어 / Español / Français / Deutsch / Português | MVP（フェーズ1） |
| **Tier 3** | العربية（RTL）/ हिन्दी / Bahasa Indonesia / Tiếng Việt / ไทย / Русский | フェーズ2以降・需要に応じて |

> まず日英の2言語で品質を固め、辞書構造を確立してから機械翻訳＋ネイティブ監修で拡張する。

### 5.2 翻訳しない設計（非言語UIが最大の武器）

座禅は本来「言葉を手放す」行為。UIの大半を**非言語**で成立させれば、翻訳コストも誤訳リスクも激減する。

- **鐘の音** — 開始・終了の合図は全言語共通。文字不要。
- **灯り／炎** — 在室＝灯りの数。人数バッジに数字（Intlで現地表記）のみ。
- **呼吸アニメーション** — 拡がる/縮む円で吸う・吐くを誘導。言語非依存。
- **姿勢イラスト** — 結跏趺坐・椅子坐・法界定印を図で。矢印とシルエットだけ。
- **アイコンと色** — 設定・言語・音量はピクトグラム。文字は補助。

### 5.3 UI文字列の国際化（技術）

- **辞書方式**: 言語ごとの JSON（`/locales/ja.json`, `en.json`, …）。キーでテキストを引く。
  ```json
  // locales/en.json
  { "hall.enter": "Enter the Hall", "bell.next": "Next bell in {min} min" }
  ```
- **言語の自動判定**: 初回は `navigator.language` で推定 → 手動切替を `localStorage` に保存。
- **切替UI**: ヘッダーに常設の言語セレクタ（地球儀アイコン＋「あ/A/中」）。ログイン不要で即切替。
- **数値・時刻・日付**: `Intl.NumberFormat` / `Intl.DateTimeFormat` で現地化（例: 在室人数、タイマー）。
- **RTL対応**: アラビア語等は `dir="rtl"` を `<html>` に付与し、CSSは論理プロパティ（`margin-inline-start` 等）で左右非依存に。
- **フォント**: 字形ごとにフォールバック指定（CJK / Arabic / Thai / Devanagari）。可変フォント＋システムフォント優先で軽量に。
- **フレームワーク**: 静的なら軽量自前ローダー、Next.js なら `next-intl` / `i18next` を採用。

### 5.4 コンテンツの多言語化

| コンテンツ | 方針 |
|---|---|
| 座禅ガイド（はじめ方） | 全Tier言語で用意。本文は短く、図中心。 |
| 音声ガイド（誘導瞑想） | 言語別に収録 or 高品質TTS。ON/OFF・速度・声選択可。 |
| 法話・提唱（フェーズ2） | 原語音声＋多言語**字幕**（VTT）。同時翻訳はしない。 |
| 利用規約・理念 | 法的に重要なため**ネイティブ監修必須**。機械翻訳のみは不可。 |
| 環境音・鐘 | 翻訳不要（共通アセット）。 |

### 5.5 文化・宗教への配慮

- **宗派・宗教中立の訳語**を選ぶ。例: "Zazen" は無理に意訳せず原語＋短い注を添える（meditation と混同させない判断も）。
- 仏教固有語（只管打坐・経行・法界定印・**調身/調息/調心**）は**原語ローマ字＋現地語の説明**を併記し、教義の押し付けにならないようにする。例: 調息 = *chōsoku*（regulating / harmonizing the breath）。
- 宗教的に敏感な地域向けに「宗教ではなく姿勢と呼吸の練習」としても入れる導線を用意。
- 右から左・縦書き文化など、レイアウトの前提を一つに固定しない。

### 5.6 翻訳の運用フロー

1. 日本語/英語を**正典（source of truth）**とする。
2. 機械翻訳（DeepL/LLM）でドラフト → **各言語ネイティブ実践者が監修**。
3. 監修者はコミュニティ「当番」から募る（座禅文化に明るい人が訳すと質が上がる）。
4. 訳の貢献を GitHub などで**オープンに管理**（誰でも修正提案できる）。
5. 未翻訳キーは英語にフォールバックし、画面が壊れないようにする。

### 5.7 やらないこと

- セッション中の**リアルタイム自動翻訳チャット**（沈黙が基本／誤訳が場を乱す）。
- 機械翻訳の**無監修公開**（特に理念・規約・宗教的文言）。
- 言語ごとにコンテンツ量を変える差別的対応（図と鐘で核体験は全言語平等に）。

## 7. 技術構成

### 7.1 MVP（小さく始める）
| 項目 | 選択肢 | 理由 |
|---|---|---|
| フロント | 静的サイト（HTML/CSS/JS）または Next.js | まず「堂」と鐘・タイマーは**サーバーなしでも動く**。 |
| 共同サイクルの同期 | クライアント側で**サーバー時刻に同期**（全員が同じ世界時計で同じ鐘を聞く） | リアルタイム通信不要。最も低コストで「一緒に座っている」を実現。 |
| 在室人数 | 軽量WebSocket（例: Supabase Realtime / Ably / 自前WS） | 人型/灯りの増減だけ。映像・音声は送らない＝**極めて軽い**。 |
| ホスティング | Vercel / Cloudflare Pages | 無料枠で開始可。 |

> **設計の肝**: 全員が「サーバー基準の同一時刻」を参照すれば、映像を中継しなくても全世界で**同じ瞬間に同じ鐘**が鳴り、「一緒に座っている」体験が成立する。帯域・コストを劇的に下げられる。

### 7.2 フェーズ2（共在感・配信）
- 在室の地図表示、当番セッション、軽量プレゼンス → **Supabase / Firebase**。
- ライブ配信が必要なら **YouTube Live + OBS**（堂の抽象映像を流す。参考動画と同じ運用）。

### 7.3 やらないこと（意図的に）
- 全員の顔/音声のリアルタイム双方向中継（重い・プライバシーリスク・「静けさ」を壊す）。
- ログイン必須化（出入りの自由を損なう）。匿名・ゲスト入堂を基本に。

### 7.4 YouTube連動運営の現実（難易度と役割分担）

結論: **技術的にはそれほど難しくない。が、YouTube単体では本企画の核は届かない。** 役割を分けて両輪にするのが現実解。

**YouTubeが得意**: 発見・集客（自習室配信のように「流れている場」に人が集まる）／一場面＋環境音を常時流すだけなら低コスト。

**YouTubeが苦手・できない**:
- **個別化・双方向がムリ** — 一人ひとり違う座処・姿勢選択・線香タイマー・多言語切替・数息観・今日の庭は出せない（全員に同じ一画面）。
- **遅延（数秒〜数十秒・視聴者ごとにバラつく）** — 鐘が厳密には同時にならない（瞑想用途なら実害小。アプリのサーバー時刻同期の方が正確で安い）。
- **24/7無停止**には常時起動エンコーダ（専用PC or クラウド）＋監視・自動再起動が必要。
- **収益化に登録者・視聴時間の条件**。音源は**著作権/Content ID**のクリア必須（→音のお布施の審査・権利処理で担保）。

**運用案（負担の軽い順）**:
1. **定時ライブ（当番会）** — 24/7にせず各タイムゾーンの決まった時刻だけ配信（暁天など）。機材最小・コミュニティが育つ。**推奨**。
2. **24hアンビエント・ループ** — 一場面をループ配信、鐘を一定間隔で差し込み、チャットはオフ。
3. **ハイブリッド** — ②を常時流しつつ①を随時。

**役割分担（確定方針）**: **YouTube＝表の入口（出会い・常時の気配）／ Web・アプリ＝堂の中（個別化された本体験）。** §3.8「両方の入口」をこの分担で運用する。

### 7.5 ホスティング／インフラ（自宅PCは使わない）【結論】

24/7を自宅PCで担うのは非現実的。**2つに分けて**解決する。

**(A) アプリ＋共在バックエンド ＝ 常時起動マシン不要（サーバーレス）**
- Google なら **Firebase Hosting ＋ Cloud Run/Functions ＋ Firestore**（無料枠大）。同等で **Cloudflare Pages / Vercel** も可。
- 設計の肝「全員がサーバー時刻に同期して同じ鐘」は静的配信＋時刻源でほぼサーバー不要。在室人数は Firestore / Supabase Realtime。
- → **自宅PC不要・スケールゼロ・ほぼ無料**。`issoku.org` をそのまま接続。

**(B) 24時間YouTube配信 ＝ 唯一“常時起動マシン”が要る部分（自宅以外で立てる具体策）**
- **A. 管理サービスに任せる（サーバー不要・最も楽）** — Restream.io / Castr 等にループ映像を上げれば24/7配信。月$10〜20程度、運用ゼロ。
- **B. クラウドVMで ffmpeg ループ（安い・自由）** —
  - **Oracle Cloud「Always Free」ARM VM**：事実上無料で24/7可（最安）。
  - 有料VPSなら Hetzner（約€4）／DigitalOcean・Vultr・Linode（$5〜）／AWS Lightsail（$5）／GCP e2-micro（無料枠・制限あり）。
  - VM上で `ffmpeg` がプリレンダ映像＋環境音を YouTube RTMP（`rtmp://a.rtmp.youtube.com/live2` ＋ストリームキー）へ送出。**systemd の `Restart=always`** で落ちても自動復帰。
- **C. そもそも24/7をやめる** — 定時の**当番ライブ**だけにすれば常時起動マシン不要。各タイムゾーンで暁天を巡らせる。

**結論**: 自宅PCの常時稼働は不要。アプリは Google/Cloudflare のサーバーレス、配信はまず当番ライブ → 必要になれば小VM/Restream。

## 8. 運営・持続可能性

- **無料 + お布施（寄付）モデル** — 寺なのでサブスクではなく任意の喜捨（Stripe/PayPal/お賽銭ボタン）。

## 10. 名称・ブランディング

### 【決定】サービス名＝一息（Issoku）／その中の寺＝坐雲堂（The Un-do）

> **入れ子の設定**: **一息（Issoku）** という世界（サービス／アプリ）があり、その中に **坐雲堂（The Un-do）** という寺がある。
> 利用者は **「一息」入れて → 「坐雲堂」に入って座る**。入口の呼吸が一息、座る堂が坐雲堂。

#### 一息（Issoku）— サービス／ブランド名

- **意味の三重**:
  1. **ひと息 ＝ 呼吸** — 発案者の原点「**調息**」に直結（§3.6 ★原点）。
  2. **ひと息つく ＝ ひと休み** — オフィスのすきま坐禅（§3.9）。
  3. **たった一呼吸に全てを込める** — 只管打坐／手放す（§1）。
- **ドメイン【決定】**: **`issoku.org`**（$9.99／年）を本ドメインに。※ `.com`・`.jp` は既に取得済みのため `.org`（寺・非営利に最適）を採用。`issoku.app` も併せて押さえると安心（任意）。取得はお名前.com で本人が実施。
- **国際表記**: "Issoku" ＋ 補足 "one breath"。
- ロゴ案: 線香から立つ**一筋の煙**、または墨の**一画**。「一息」そのものが絵になる。

#### 坐雲堂（The Un-do）— その中の寺（メインルーム「堂」の名・§3.1）

> 書けば英語「The Un-do」、声に出せば日本語「坐雲堂（ざ・うんどう）」。冠詞 "The" が日本語では「坐（ざ）」になる。

- **由来（四重の掛けことば）**: ①**The（ザ）＝坐**（坐禅）②**Un-do＝雲堂（うんどう）**（僧堂／クラウド）③**undo**（取り消す）④**un-do＝し-ない＝手放す**（§1の背骨）。
- 公開ブランド・検索・ドメインは **一息（issoku）**＝既存アプリ「雲堂(undo)」と衝突しない・埋もれない。坐雲堂（The Un-do）は **寺の名（世界観・愛称）として内側に保持** → 四重の言葉遊びと坐雲堂の魂を失わない。**前回の衝突懸念は、坐雲堂を“表のブランド”から“中の寺名”へ移したことで解消。**

### 設計への含意（重要・変わらず）

線香タイマー＋音声オリエン＋鐘3回は既存「雲堂」アプリにもある＝**目玉ではない**。本企画の独自性＝**24時間・国際/多言語・多彩な座処・世界との共在（地図の灯り）・音のお布施・椅子座禅・YouTube表口×アプリ本体の両輪**を前面に出す。

### 参考：見送った代替案（ドメイン空き）

円相（Ensō, `ensohall.org`）/ 暁天（Gyōten, `gyoten.org`）/ The Open Hall（`theopenhall.org`）。

- トーン: 余白の多い和モダン。墨・生成り・灯りの橙。アニメーション最小、動きは穏やかに。
